Повідомлення від Тім Ліда (Олександр Степаненко):

Огляд проекту:
Створення платформи для обміну документами між підприємствами, яка забезпечує конфіденційність даних та зручність використання. Платформа дозволяє користувачам безпечно обмінюватися документами та зберігати історію доступу. Вона повинна включати надійний бекенд для обробки документів, API-сервіс для інтеграції з іншими системами та простий статичний фронтенд для зручності користування.

Задача:
Розгорнути інфраструктуру з бекендом на ECS для обробки запитів користувачів, налаштувати зберігання документів та даних користувачів у RDS, а для швидкого доступу до інформації – застосувати Redis. Використовувати S3 для зберігання статичних файлів і CloudFront для кешування, що прискорить завантаження контенту. При виконанні завдання студенти мають спиратись на надану архітектурну діаграму для правильної реалізації всіх компонентів системи.

База даних:
RDS (PostgreSQL) використовується для зберігання документів та історії доступу. Redis для кешування частих запитів, що підвищує продуктивність системи і забезпечує швидкий доступ до даних.

Докеризація:
Усі сервіси мають бути докеризовані для забезпечення легкої масштабованості та зручного розгортання. Це спростить керування та дозволить легко розширювати функціонал системи.

CI/CD процес:
Налаштувати ефективний CI/CD процес для автоматизації деплою додатку, з окремими пайплайнами для деплою бекенду і фронтенду, щоб забезпечити стабільність роботи та швидкість оновлень.

Важливі нотатки:
Інфраструктура повинна бути налаштована на обробку 10 запитів за секунду під час звичайного навантаження і до 25 запитів за секунду під час пікових навантажень. Загальний бюджет для інфраструктури не повинен перевищувати $175 на місяць.
